Репозиторий содержит шесть лабораторных работ по WinAPI и синхронизации. В коде используются CMake, обработка ошибок Windows API через `GetLastError`/`FormatMessage`, исключения C++, инициализация структур перед использованием и сообщения на английском языке.

## Сборка

Лабораторные работы 1-5 используют Windows API, поэтому собираются на Windows:

```powershell
cmake -S . -B build -G "Visual Studio 17 2022"
cmake --build build --config Release
```

Можно собирать отдельную лабораторную:

```powershell
cmake -S lab1 -B build-lab1
cmake --build build-lab1 --config Release
```

Лабораторная 6 написана на Java и может быть собрана отдельно:

```bash
javac -encoding UTF-8 -d build/lab6/classes lab6/src/main/java/lab3/*.java
java -cp build/lab6/classes lab3.Main
```

## Краткое описание

### Лабораторная работа 1

`Main` запускает `Creator` и `Reporter` через `CreateProcess`, ожидает завершения дочерних процессов через `WaitForSingleObject`, выводит бинарный файл с сотрудниками и итоговый текстовый отчет. `Reporter` сортирует сотрудников по имени и рассчитывает зарплату по ставке за час.

### Лабораторная работа 2

Программа создает два потока: `min_max` ищет минимум и максимум с задержкой 7 мс после сравнений, `average` считает среднее значение с задержкой 12 мс после суммирования. После завершения потоков `main` заменяет минимальные и максимальные элементы средним значением.

### Лабораторная работа 3

Несколько потоков `marker` заполняют общий массив своими номерами. При невозможности продолжить работу каждый поток сообщает `main`, после чего `main` по очереди завершает выбранные потоки, очищает их метки и разрешает остальным продолжить работу.

### Лабораторная работа 4

`Receiver` и несколько процессов `Sender` обмениваются сообщениями через общий бинарный файл, организованный как кольцевая FIFO-очередь. Для синхронизации используются именованные mutex, semaphore и event.

### Лабораторная работа 5

`Server` хранит записи сотрудников и обслуживает процессы `Client` через named pipes. Для каждой записи используется readers-writer lock: несколько клиентов могут читать одну запись одновременно, но изменение выполняется эксклюзивно.

### Лабораторная работа 6

Java-реализация задачи из лабораторной 3. Функциональность сохранена, но синхронизация переделана на `CountDownLatch` и `Semaphore`, что убирает риск потерянных уведомлений при ожидании состояния `stuck`.
